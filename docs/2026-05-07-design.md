# mp-pdf-bot — 设计

公众号文章自动归档为 PDF 的小工具。基于 wewe-rss + headless Chrome。

## 目标场景

- 用户：作者本人 + 朋友同事可拿去用
- 平台：仅 macOS
- 一台机一份独立部署，朋友扫自己的微信读书

## 用户操作流程

1. `git clone` 仓库到本地
2. `./mybot setup` — 一次性安装（依赖检查 / 起容器 / 加 cron）
3. `./mybot open` — 浏览器打开 wewe-rss 后台，扫码登录微信读书 + 添加公众号
4. 之后无人值守。每天 18:00 自动 sync，PDF 落到 `./pdfs/<公众号名>/<日期>-<标题>.pdf`
5. 想立即拉一次：`./mybot sync`

## 仓库结构

```
mp-pdf-bot/
├── README.md           # 安装 + 用法 + 排错
├── mybot               # 主入口（python，shebang 直接执行）
├── config.example.env  # 配置模板，setup 复制为 config.env
├── .gitignore          # 忽略 config.env / pdfs/ / wewe-rss-data/
└── docs/               # 设计文档
```

## 子命令

| 命令 | 行为 |
|------|------|
| `./mybot setup`     | 校验环境、起容器、加 cron、生成 config.env |
| `./mybot sync`      | 读 SQLite 找新文章 → headless Chrome 转 PDF |
| `./mybot open`      | 浏览器打开 `http://localhost:<port>/dash` |
| `./mybot logs`      | `docker logs -f wewe-rss` |
| `./mybot stop`      | 停容器（cron 保留） |
| `./mybot start`     | 启容器 |
| `./mybot uninstall` | 移除容器 + cron。数据目录保留，提示路径 |

## 配置（config.env）

```
WEWE_RSS_PORT=4000
AUTH_CODE=mp-pdf-bot                 # 默认固定，朋友自己想改去改
PDF_OUTPUT_DIR=./pdfs                # 相对仓库根目录
WEWE_RSS_DATA_DIR=./wewe-rss-data    # SQLite 持久化卷
CONTAINER_NAME=mp-pdf-bot-wewe-rss   # 加前缀避免和别人的 wewe-rss 撞名
CRON_TIME="0 18 * * *"
```

## 关键实现要点

**Chrome 路径自动发现**：按顺序尝试
1. `/Applications/Google Chrome.app/Contents/MacOS/Google Chrome`
2. `/Applications/Chromium.app/Contents/MacOS/Chromium`
3. `/Applications/Microsoft Edge.app/Contents/MacOS/Microsoft Edge`

都没有则报错，提示用户装 Chrome。

**SQLite 路径**：`<仓库根>/wewe-rss-data/wewe-rss.db`（容器挂卷过来）。脚本不进容器查 DB，直接读宿主机文件。

**Docker 检查**：
- `docker info` 失败 → 提示 "请打开 Docker Desktop"
- 没装 Docker → 提示 "brew install --cask docker"
- 不自动装 Docker（每个人偏好不同，别替朋友决策）

**Cron 装法**：
- 读现有 `crontab -l`
- 找含 `# mp-pdf-bot` 注释的行 → 删掉
- append 新行：`<CRON_TIME> cd <仓库绝对路径> && ./mybot sync >> sync.log 2>&1 # mp-pdf-bot`
- `crontab -` 写回

**PDF 文件名清理**：保留中文，把 `/ \ : * ? " < > |` 和控制字符替换为 `_`，截断 80 字符。

**幂等**：sync 跳过已存在的 PDF（按文件名匹配）。setup 重跑安全（容器存在时先 stop+rm）。

## 不做的事（YAGNI）

- 不支持 Linux / Windows
- 不做 Web UI（用 wewe-rss 自带的）
- 不做 CLI 添加公众号（同上）
- 不打包 Chrome 进容器（PDF 渲染走宿主机 Chrome，最简单且效果最好）
- 不做账号池 / 多用户（一台机一个微信读书账号）
- 不做 PDF 去水印 / 美化排版（Chrome 默认打印就行）
- 不做失败重试机制（cron 每天跑，下次自然补上）
