# mp-pdf-bot — 设计

公众号文章自动归档为 PDF 的小工具。基于 wewe-rss（Node.js 本地跑）+ headless Chrome。

## 目标场景

- 用户：作者本人 + 朋友同事可拿去用
- 平台：仅 macOS
- 一台机一份独立部署，朋友扫自己的微信读书

## 用户操作流程

1. `git clone` 仓库到本地
2. `./mybot setup` — 一次性安装（依赖检查 / 拉 wewe-rss 源码 / 构建 / 装 launchd / 加 cron）
3. `./mybot open` — 浏览器打开 wewe-rss 后台，扫码登录微信读书 + 添加公众号
4. 之后无人值守。每天 18:00 自动 sync，PDF 落到 `./pdfs/<公众号名>/<日期>-<标题>.pdf`
5. 想立即拉一次：`./mybot sync`

## 仓库结构

```
mp-pdf-bot/
├── README.md           # 安装 + 用法 + 排错
├── mybot               # 主入口（python，shebang 直接执行）
├── config.example.env  # 配置模板，setup 复制为 config.env
├── .gitignore
├── docs/
│   └── design.md
└── (运行时生成，git 忽略)
    ├── wewe-rss/       # setup 时 git clone 进来
    ├── data/           # SQLite DB 持久化
    │   └── wewe-rss.db
    ├── pdfs/           # PDF 输出
    ├── logs/           # wewe-rss / sync 日志
    └── config.env
```

## 子命令

| 命令 | 行为 |
|------|------|
| `./mybot setup`     | 校验依赖 → clone 源码 → build → 装 launchd → 加 cron → 生成 config.env |
| `./mybot sync`      | 读 SQLite 找新文章 → headless Chrome 转 PDF |
| `./mybot open`      | 浏览器打开 `http://localhost:<port>/dash` |
| `./mybot logs`      | tail wewe-rss + sync 日志 |
| `./mybot stop`      | `launchctl bootout` 停服务（cron 保留） |
| `./mybot start`     | `launchctl bootstrap` 启服务 |
| `./mybot status`    | 显示 launchd 状态 + HTTP 健康检查（curl localhost:port）+ DB 大小 + PDF 数量 |
| `./mybot uninstall` | bootout launchd + 删 plist + 清 cron。`./wewe-rss/` 删除，`./data/` 和 `./pdfs/` 保留 |

## 依赖检查（setup 时按顺序）

| 依赖 | 缺失时行为 |
|------|------------|
| macOS                | 报错退出 |
| Homebrew             | 报错，给安装链接，不自动装 |
| Node.js ≥ 20         | 提示 `brew install node`，不自动装（朋友可能想用 fnm/nvm） |
| pnpm                 | 自动 `corepack enable && corepack prepare pnpm@latest --activate`（Node 自带） |
| Google Chrome / Edge | 报错，提示装 Chrome |

## 配置（config.env）

```
WEWE_RSS_PORT=4000
AUTH_CODE=mp-pdf-bot
PDF_OUTPUT_DIR=./pdfs
DATA_DIR=./data
WEWE_RSS_REPO_REF=v2.6.1           # 锁定 wewe-rss 版本，setup 时 git checkout 到此 tag
CRON_TIME="0 18 * * *"
```

## 关键实现要点

### wewe-rss 安装（setup 内部步骤）

1. `git clone https://github.com/cooderl/wewe-rss.git ./wewe-rss && cd ./wewe-rss && git checkout $WEWE_RSS_REPO_REF`
2. SQLite schema 切换：`rm -rf apps/server/prisma && mv apps/server/prisma-sqlite apps/server/prisma`
3. 写 `./wewe-rss/.env`：`DATABASE_URL=file:../data/wewe-rss.db`, `AUTH_CODE`, `FEED_MODE=fulltext`, `SERVER_ORIGIN_URL=http://localhost:$PORT`
4. `pnpm install`
5. `npx prisma generate --schema apps/server/prisma/schema.prisma`
6. `npx prisma migrate deploy --schema apps/server/prisma/schema.prisma`
7. `pnpm run -r build`

### 服务管理：launchd

- plist 模板生成到 `~/Library/LaunchAgents/com.mp-pdf-bot.wewe-rss.plist`
- 路径全部用绝对路径（仓库根 + node 二进制）
- WorkingDirectory 设为 `<仓库根>/wewe-rss`
- ProgramArguments：`pnpm run start:server`
- StandardOutPath / StandardErrorPath：`<仓库根>/logs/wewe-rss.log`
- KeepAlive 开（崩了自动重启）
- RunAtLoad 开（开机自启）
- `launchctl bootstrap gui/$UID <plist>` 安装并启动
- `launchctl bootout gui/$UID/com.mp-pdf-bot.wewe-rss` 停止

### Chrome 路径自动发现

按顺序尝试：
1. `/Applications/Google Chrome.app/Contents/MacOS/Google Chrome`
2. `/Applications/Chromium.app/Contents/MacOS/Chromium`
3. `/Applications/Microsoft Edge.app/Contents/MacOS/Microsoft Edge`

都没有则报错。

### Cron

- `crontab -l` 输出
- 删除含 `# mp-pdf-bot` 注释的行
- append：`<CRON_TIME> cd <仓库绝对路径> && ./mybot sync >> logs/sync.log 2>&1 # mp-pdf-bot`
- `crontab -` 写回

### PDF 文件名清理

保留中文，把 `/ \ : * ? " < > |` 和控制字符替换为 `_`，截断 80 字符。

### 幂等性

- setup 重跑安全：launchd 已存在则先 bootout，wewe-rss/ 已存在则跳过 clone（除非 `--reinstall`）
- sync 跳过已存在的 PDF（按文件名匹配）

## 不做的事（YAGNI）

- 不支持 Linux / Windows
- 不做 Web UI（用 wewe-rss 自带的）
- 不做 CLI 添加公众号（同上）
- 不做账号池 / 多用户
- 不做 PDF 去水印 / 美化排版
- 不做失败重试（cron 每天跑，下次自然补上）
- 不做自动 Node/Chrome 安装（替朋友决策不合适）
- 不做 wewe-rss 自动升级（手动 `./mybot update` 可后续加，本期不做）
